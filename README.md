# Policy-as-Code Engine

A lightweight **Policy-as-Code Framework** for evaluating security rules using YAML-based policies.

This project demonstrates how an enterprise can:

- Encode security requirements as code  
- Evaluate configurations automatically  
- Generate human-readable reports  
- Integrate security into CI/CD workflows  

Perfect for CISOs, security architects, and DevSecOps leaders who want to showcase modern security engineering capabilities.

---

##  Features

- YAML-based security policies  
- Simple evaluation engine  
- JSON report generator  
- Rich CLI visualization  
- Extensible design  

---

##  Example Use Cases

- Cloud configuration compliance  
- DevSecOps enforcement  
- CIS benchmark checks  
- Internal security policy validation
